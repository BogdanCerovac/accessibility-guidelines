Web Content Accessibility Guidelines Summary

This document will help you quickly get up to speed with the Web Content Accessibility Guidelines 2.1, and avoid common accessibility-related mistakes. It covers all A- and AA-level requirements.

This summary is a simplification. If you're not sure how to meet a requirement, check the official Web Content Accessibility Guidelines 2.1.

1. Easy to perceive

Your website/app must present information in ways that people can recognise and use, no matter how they consume content (for example if they have low vision, or use assistive technologies like screen readers).

Provide text alternatives for images

  • Text alternatives for images. Provide an alternative text description for images. Make sure the alternative description conveys the same message or functionality.

Provide alternatives for audio content, videos and presentations

  • Transcript (for prerecorded audio-only content). For audio content that has no video (like a podcast), provide a transcript.
  • Captions (for prerecorded videos and presentations). Provide captions for videos and presentations that are pre-recorded.
  • Text or audio description (for prerecorded videos and presentations). For videos and presentations that are pre-recorded, provide an equivalent text description, or an audio description of what happens on screen.
  • Captions (for live videos and presentations). Provide captions for videos and presentations that are performed live.
  • Audio description (for prerecorded videos and presentations). For videos and presentations that are pre-recorded, provide an audio description of what happens on screen, even if you've already provided a text description.

    • Create content that can be presented in different ways

  • Structure and relationships. When designs convey a structure visually (like distinct sections within a page, or a label next to a checkbox) that structure must also be identified in code.
